# 📝 CilION Implementation Plan (To-Dos)

## Phase 1: Environment Setup & Scaffolding
*Goal: Get the basic tools installed and a native-routing Kubernetes environment running.*

- [x] **Initialize the Project:**
  - Run `go mod init github.com/martenwallewein/cilion`.
  - Create the directory structure using Kubebuilder (`kubebuilder init`).
- [x] **Install eBPF Toolchain:**
  - Install `clang`, `llvm`, `libbpf-dev`, and `bpftool` on your Linux development machine.
- [x] **Setup Kubernetes with Native Routing (KinD / Hetzner):**
  - Spin up a multi-node cluster.
  - Install Cilium in **Native Routing Mode** (`tunnel=disabled`, `autoDirectNodeRoutes=true`).
  - Configure cloud fabric routing (e.g., Hetzner static routes) to allow cleartext Pod IPs to traverse the L3 network.
- [ ] **Setup Local SCION Topology:**
  - Clone the official `scionproto/scion` repository. Add as submodule.
  - Run their local topology generator to create a simulated 3-AS network on your host machine.
- [x] **Makefile eBPF Compilation:**
  - `make generate` — triggers `bpf2go` to compile C code into eBPF objects and generate Go wrappers.

## Phase 2: The eBPF Data Plane (Underlay Interception)
*Goal: Successfully wrap a cleartext Pod packet in a SCION header at the physical interface.*

- [ ] **Define C Headers (`bpf/headers/scion.h`):**
  - Write the C structs for the SCION Common Header, Address Header, and Path Header.
- [ ] **Define eBPF Maps (`bpf/cilion_maps.c`):**
  - Create the `pod_policy_map` (Key: Pod IP, Value: Policy ID).
  - Create the `scion_path_cache` (Key: Policy ID, Value: Raw SCION Header Bytes & Next-Hop IP).
- [ ] **Write TC Egress Program (`bpf/tc_egress.c`):**
  - Hook to the **physical interface** (e.g., `eth0`), *not* the Pod `veth`.
  - Parse the Ethernet and IPv4 headers. Look for source IPs belonging to the K8s Pod CIDR.
  - Implement a map lookup using the cleartext source IP.
  - Use `bpf_skb_adjust_room()` to expand the packet buffer.
  - Inject a hardcoded SCION-over-UDP header (for initial testing).
  - Recalculate outer IP/UDP checksums.
  - Change the outer destination IP to the SCION Border Router / Transit Provider.
- [ ] **Write XDP / TC Ingress Program (`bpf/xdp_ingress.c`):**
  - Intercept UDP port 30041 on the **physical interface**.
  - Strip the outer IP, UDP, and SCION headers.
  - Pass the inner IP packet up the stack so Cilium can process the ingress policy.
- [ ] **Manual Datapath Verification (The MVP):**
  - Load the eBPF programs manually via Go or `bpftool`.
  - Ping from Pod A to Pod B and verify via `trace_pipe` and `tcpdump` that packets leave the physical interface wrapped in SCION-UDP.

## Phase 3: The Go Control Plane (Intent vs. State via CRDs)
*Goal: Implement the Kubebuilder operators to separate external API calls from local node injection.*

- [x] **Generate eBPF Go Bindings:**
  - Run `go generate` to emit typed wrappers for the `scion_path_cache` and `pod_policy_map`.
- [ ] **Create the CRDs (`api/v1alpha1`):**
  - `ScionPathPolicy` (User Intent)
  - `ScionComputedPath` (System State - generated by the controller)
  - `ScionLink` (Uplink configuration)
- [ ] **Build the Global Controller (`pkg/controller/global`):**
  - Runs as a standard Deployment.
  - Watches `ScionPathPolicy`.
  - Talks to the SCION Control Service (Phase 4).
  - Creates/Updates/Deletes `ScionComputedPath` CRDs.
- [ ] **Build the Local Node Agent (`pkg/controller/agent`):**
  - Runs as a privileged DaemonSet on every worker node.
  - Attaches `TC` and `XDP` programs to the host's physical interface.
  - Watches `ScionComputedPath` CRDs and populates the `scion_path_cache` eBPF map.
  - Watches local `Pod` scheduling (filtered by `spec.nodeName`) and populates the `pod_policy_map`.

## Phase 4: SCION Integration
*Goal: Connect the Global Controller to the SCION network to fetch dynamic paths.*

- [ ] **Implement the SCION Client (`pkg/scion`):**
  - Import `github.com/scionproto/scion/pkg/daemon`.
  - Connect the Global Controller to the `sciond` API.
  - Fetch available paths to remote Autonomous Systems.
- [ ] **Path Translation (`pkg/datapath`):**
  - Write Go logic inside the Global Controller to serialize a `snet.Path` into the exact raw byte array required by the C `scion_path_entry` struct.
  - Write this raw byte array into the `ScionComputedPath.Spec.HopFields`.
- [ ] **Implement App-Aware Routing logic:**
  - Filter the paths fetched from `sciond` based on `ScionPathPolicy` constraints (e.g., shortest path, avoid specific ISDs).

## Phase 5: Advanced eBPF & Cilium Coexistence
*Goal: Handle the hardest technical hurdles for a production-ready, side-by-side datapath.*

- [ ] **eBPF Chaining (TC eXpress):**
  - Update the Local Agent to use `tcx` (or strict priority attachment) to ensure CilION's egress program runs strictly *after* Cilium's egress program on the physical interface.
- [ ] **Dynamic AES-MAC Generation:**
  - SCION requires a MAC to validate hop fields.
  - *Option A (Static):* Pre-compute the MAC in the Global Controller and push it via `ScionComputedPath`.
  - *Option B (Dynamic):* Implement AES-CMAC directly in eBPF C code using `bpf_crypto_ctx` to compute it per-packet.
- [ ] **MTU Clamping (TCP MSS):**
  - Update `tc_egress.c` to parse TCP SYN packets and modify the TCP MSS option down (e.g., 1350 bytes) to prevent WAN fragmentation caused by the SCION overhead.
- [ ] **Cilium Identity Embedding (Optional / Research):**
  - Modify the datapath to append a SCION End-to-End (E2E) Extension header.
  - Read Cilium's `skb->mark` or identity map and embed the Cilium Security Identity into the SCION header on Egress, extracting it on Ingress.
